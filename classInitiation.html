void Initiation::VolumeMass ( Hydrodynamics &  hydro,
ParticleManager &  particles,
spKernel  weight_function 
)

predict the particle volume and mass

mass initiation is different from 1DSPH code: here: mass is calculated by summing up the kernel function contributions for easch particle, which gives a kind of the inverse volume taken by each particle (not perfectly true at the discontinuity). together with rho (from initialization) a mass for each particle can be obtained. within the discontinuity zone, this mass varies because of the smoothing effect of the kernel summation. The mass for each particle stays constant during the simuation.

  • iterate particles on the particle list

    • pick an origin particle

      size of the list can be zero in some circumstances but in 1D (2D) shock it is not expected

    • iterate this Nearest Neighbor spParticle list

    • calculate distance (origin<-> neighbor)

    • calculate weight function for given distance (w=0, if dist>supportlengtg) an summ it up

  • calculate volume as reciprocal value of weight function

  • save volume and mass in the respective particle list node (whih is each a spParticle object with all the particle properties)
